---
title: 'Reverse Reward Transaction'
description: 'Endpoint to reverse a reward transaction'
sidebarTitle: 'Reverse Transaction'
api: 'POST https://rewardservice.withbenji.com/transactions/{id}/reverse'
---

<Note>The Bearer token to be used on this API is your **partner access token**</Note>

<ParamField path="id" type="integer" required>
  The Benji Platform ID of the reward transaction
</ParamField>

### Return values 
<ResponseField name="reward_transaction" type="object" >
  A reward_transaction object property
  <Expandable title="properties" defaultOpen="true">
    <ResponseField name="reward_transaction_id" type="integer">Benji Platform reward transaction identifier</ResponseField>
    <ResponseField name="transaction_status" type="string">
    String representing one of the transaction status values : 
        <Expandable title="possible transaction status values" type="enum">
            - **REVERSED (1)** : Indicating a transaction was successfully reversed <br/>
            - **FAILED (2)** : Indicating a transaction failed to reverse since it has already been completed  <br/>
            - **PENDING (3)** : Indicating a transaction is pending and has not been reversed yet  <br/>
            - **INITIATED (4)** : Indicating a transaction was initiated but not yet reversed  <br/>
        </Expandable>
    </ResponseField>
    
  </Expandable>
</ResponseField>
